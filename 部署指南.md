# 锤锤的备考冒险 — 部署 & 云端升级指南

这份文档教你从**本地版**升级到**云端同步版**,让你和孩子各自打开 URL 都能看到同一份数据。

---

## 第一步:本地试用(0 分钟,立刻可用)

最简单的方式 — **直接双击 `锤锤的备考冒险.html`**,浏览器就会打开。

### 用法
- 主页:看锤锤当前形态、进度条、装备
- 打卡:每周日完成复盘后勾选(自动加 5 分)
- 历史:看 26 周积分曲线
- 管理:家长加分扣分(老师反馈、月小测达标等)

### 注意
- 数据存在浏览器,**只能在打开的设备上看**
- 关掉浏览器再打开,数据还在
- **建议每周用「导出数据」按钮备份一次 JSON**

---

## 第二步:部署到 GitHub Pages(15 分钟,有公网 URL)

让你和孩子都能通过 URL 访问,但**数据还是本地**(各自的设备数据独立)。

### 操作步骤

1. **注册 GitHub 账号**(如果还没有)
   - 访问 https://github.com/signup
   - 用邮箱注册

2. **创建仓库**
   - 登录后点右上角 `+` → `New repository`
   - 仓库名:`chamui-psle`(或你喜欢的)
   - 勾选 `Public`(公开,免费)
   - 点 `Create repository`

3. **上传 HTML 文件**
   - 在仓库页面点 `uploading an existing file`
   - 把 `index.html` 拖进去(注意一定要叫 `index.html`)
   - 滚到底,点 `Commit changes`

4. **开启 Pages**
   - 仓库页面 → `Settings` → 左侧 `Pages`
   - `Source` 选 `Deploy from a branch`
   - `Branch` 选 `main` / `/(root)`
   - 点 `Save`

5. **等 1-2 分钟**,会显示一个 URL,类似:
   ```
   https://你的用户名.github.io/chamui-psle/
   ```

6. **完成!**这个 URL 你和孩子都能访问。

---

## 第三步:升级到 Firebase 云同步(30-45 分钟,数据共享)

让你在新加坡看孩子打卡情况、孩子在家也能看积分,**数据真正同步**。

### 准备工作

1. **注册 Firebase**
   - 访问 https://console.firebase.google.com/
   - 用 Google 账号登录(免费)

2. **创建项目**
   - 点 `Create a project` (创建项目)
   - 项目名:`chamui-psle`
   - 不需要 Google Analytics(取消勾选)
   - 点 `Create project`

3. **添加 Web 应用**
   - 项目首页点 `</>` 图标(添加 Web 应用)
   - 应用昵称:`chamui-app`
   - 不需要 Hosting,直接 `Register app`
   - **复制 firebaseConfig 配置**(类似下面的)
   ```javascript
   const firebaseConfig = {
     apiKey: "AIzaSy...",
     authDomain: "chamui-psle.firebaseapp.com",
     projectId: "chamui-psle",
     storageBucket: "chamui-psle.appspot.com",
     messagingSenderId: "123456789",
     appId: "1:123456789:web:abc..."
   };
   ```

4. **启用 Firestore 数据库**
   - 左侧菜单 `Build` → `Firestore Database`
   - 点 `Create database`
   - 选 `Start in test mode`(测试模式,30 天免登录可写)
   - 选最近的区域(亚洲选 `asia-southeast1` Singapore)

### 修改代码

在 `index.html` 里找到 `<script>` 块的开头(`const CHAMUI = {` 这一行之前),**替换 `data.js` 部分的 `loadState` 和 `saveState` 函数**为下面的 Firebase 版:

```javascript
// === Firebase 配置(粘贴你刚才复制的) ===
const firebaseConfig = {
  apiKey: "你的 apiKey",
  // ... 其他字段
};

// 加载 Firebase SDK
const fbScript1 = document.createElement('script');
fbScript1.src = 'https://www.gstatic.com/firebasejs/10.7.0/firebase-app-compat.js';
document.head.appendChild(fbScript1);

const fbScript2 = document.createElement('script');
fbScript2.src = 'https://www.gstatic.com/firebasejs/10.7.0/firebase-firestore-compat.js';
document.head.appendChild(fbScript2);

// 等 SDK 加载完成
function waitForFirebase(callback) {
  if (typeof firebase !== 'undefined') callback();
  else setTimeout(() => waitForFirebase(callback), 100);
}

// 替换 loadState 和 saveState
async function loadState() {
  return new Promise(resolve => {
    waitForFirebase(async () => {
      firebase.initializeApp(firebaseConfig);
      const db = firebase.firestore();
      
      try {
        const doc = await db.collection('chamui').doc('main').get();
        if (doc.exists) {
          resolve(Object.assign(getDefaultState(), doc.data()));
        } else {
          resolve(getDefaultState());
        }
      } catch (e) {
        console.error('Firebase 加载失败,降级到 localStorage', e);
        const raw = localStorage.getItem(STORAGE_KEY);
        if (raw) resolve(Object.assign(getDefaultState(), JSON.parse(raw)));
        else resolve(getDefaultState());
      }
    });
  });
}

async function saveState(state) {
  // 同时存本地备份
  localStorage.setItem(STORAGE_KEY, JSON.stringify(state));
  
  // 写到 Firestore
  if (typeof firebase !== 'undefined' && firebase.firestore) {
    try {
      await firebase.firestore().collection('chamui').doc('main').set(state);
      return true;
    } catch (e) {
      console.error('Firebase 写入失败', e);
      return false;
    }
  }
  return true;
}
```

### 重新部署

把改好的 `index.html` 重新上传到 GitHub 仓库即可(覆盖原文件)。

### 测试

在你的设备打开 URL,加 5 分,然后让孩子在他的设备打开同一个 URL,刷新一下应该能看到 5 分。

## 第四步:安全加固(可选)

测试模式 30 天后会变只读。要保持长期可写,做下面 2 件事:

1. **添加用户认证**
   - 在 Firebase Console 启用 Authentication → Email/Password
   - 给爸妈和孩子各注册一个账号
   - 改写代码加登录页

2. **更新 Firestore 规则**
   - Firestore Database → Rules
   - 改成下面这样:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /chamui/{document=**} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```

如果不想搞这么复杂,**简单的做法**是每 30 天去 Firebase Console 把规则有效期延长一次。

## 常见问题

### Q: Firebase 真的免费吗?
是的,免费额度对个人使用绰绰有余:
- Firestore:每天 50K 读、20K 写、1GB 存储
- 一个家庭一天的打卡撑死 50 次操作
- 完全用不完免费额度

### Q: 数据会丢吗?
- 本地 localStorage:清浏览器数据会丢 → **每周导出 JSON 备份**
- Firebase:数据存在 Google 服务器,不会丢
- 双层保险:Firebase + 每周导出 JSON

### Q: 孩子能改自己的积分吗?
现在版本里**孩子能看到管理面板**(因为没加权限)。如果想限制孩子只能用打卡功能、不能用管理页:
- 简单:把"管理"标签隐藏(改 CSS `display: none`)
- 复杂:用 Firebase Auth 区分父母/孩子角色,展示不同 UI
